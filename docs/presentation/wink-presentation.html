WINK

Weak Incentives for Agents

A Python library for building agents as
typed, testable, deterministic objects

What is WINK?

  • Python library for building LLM agents
  • Not a workflow engine or framework
  • Focuses on the parts that benefit from determinism
  • Provider-agnostic: works with OpenAI, LiteLLM, Claude

Core Philosophy

"Weak Incentives"

What are Weak Incentives?

Shape prompts, tools, and context so the model's easiest path is also the correct one.


  • Don't fight the model - guide it
  • Make good behavior the path of least resistance
  • Use structure, not constraints

Definition vs Harness

Agent Definition (you own)

  • Prompt
  • Tools
  • Policies
  • Feedback

Execution Harness (runtime)

  • Planning loop
  • Sandboxing
  • Retries
  • Crash recovery
  • Deadlines & budgets

The harness keeps changing; your agent definition should not.

Policies Over Workflows

Aspect Workflow Policy
Specifies Steps to execute Constraints to satisfy
On unexpected Fails or branches Agent reasons
Composability Sequential coupling Independent conjunction
Agent role Executor Reasoner

Five Key Principles

  1. Prompts as First-Class Objects
    Not strings - typed immutable graphs
  2. Explicit Side Effects
    Only tool handlers modify external state
  3. Typed Contracts Everywhere
    Dataclasses for params, outputs, state
  4. Deterministic Inspection
    Same inputs ‚Üí same outputs
  5. Provider Agnostic
    Same agent, different models

Prompts

PromptTemplate vs Prompt

PromptTemplate

  • Immutable definition
  • Section tree structure
  • Declares parameters
  • Declares tools

Prompt

  • Runtime binding
  • Parameters filled in
  • Overrides applied
  • Ready for execution

Sections

Building blocks of prompts

  • Markdown content with placeholders
  • Conditional enabling
  • Progressive disclosure (FULL / SUMMARY)
  • Tools attachment
  • Compose into trees

Example: Defining a Prompt


@PromptTemplate(
    ns="myagent",
    key="code-review",
    sections=(
        Section(
            key="role",
            content="You are a code reviewer.",
        ),
        Section(
            key="instructions",
            content="Review the following code:\n\n{code}",
        ),
    ),
)
class CodeReviewPrompt:
    code: str

Tools

Typed contracts with transactional execution

Tool Structure


@dataclass(frozen=True, slots=True)
class SearchParams:
    query: str
    max_results: int = 10

@dataclass(frozen=True, slots=True)
class SearchResult:
    matches: list[str]

def search_handler(
    params: SearchParams,
    *,
    context: ToolContext
) -> ToolResult[SearchResult]:
    # Implementation here
    return ToolResult.ok(
        SearchResult(matches=["..."]),
        message="Found 5 matches"
    )

Transactional Execution

  • Every tool call is wrapped in a transaction
  • On success: changes committed
  • On failure: session state + filesystem rolled back
  • No partial state - enables safe retries

Tool Policies

Declarative constraints on tool usage


# Must read before write
policy = ToolPolicy(
    requires={"write_file": ["read_file"]},
)

# Sequential dependencies
policy = ToolPolicy(
    sequential=[("analyze", "implement", "test")],
)

Tool Suites

Bundle instructions + tools + session state

  • Sections that declare tools
  • Instructions colocated with capabilities
  • Reusable across prompts
  • Example: filesystem tools, git tools, search tools

Sessions

Deterministic agent memory

Event-Driven State

  • Slices: Immutable tuples keyed by dataclass type
  • Reducers: Pure functions that process events
  • Events: All mutations flow through dispatch

current_state + event ‚Üí new_state

Reducer Example


@dataclass(frozen=True, slots=True)
class FileRead:
    path: str
    content: str

@dataclass(frozen=True, slots=True)
class FilesReadReducer:
    @reducer(on=FileRead)
    def on_file_read(self, event: FileRead) -> SliceOp[FileRead]:
        return SliceOp.append(event)

# Usage
session.dispatch(FileRead(path="/foo.py", content="..."))
files = session[FileRead].all()

Why This Matters

  • Deterministic: Replay events, get same state
  • Inspectable: Full history of what happened
  • Testable: Pure reducers are easy to test
  • Snapshots: Save and restore for rollback

Progressive Disclosure

Managing context size dynamically

How It Works

  1. Sections start collapsed (summary only)
  2. Model requests expansion via open_sections()
  3. AgentLoop catches VisibilityExpansionRequired
  4. Applies overrides and retries

Example


Section(
    key="large-context",
    content="... lots of detailed content ...",
    summary="Summary: Available context about X",
    visibility=Visibility.SUMMARY,  # Start collapsed
)

# Model calls open_sections(["large-context"])
# Content expands to FULL visibility

Benefits

  • Keep initial prompts lean
  • Model requests what it needs
  • Automatic token management
  • Better context utilization

Production Pattern

AgentLoop


class MyAgent(AgentLoop[Request, Response]):
    def prepare(
        self, request: Request
    ) -> tuple[Prompt, Session]:
        prompt = MyPrompt(code=request.code).bind()
        session = Session()
        return prompt, session
  • Handles request-response lifecycle
  • Automatic progressive disclosure
  • Enforces deadlines and budgets
  • Manages resource lifecycle

EvalLoop

Run evaluation alongside production


eval_loop = EvalLoop(
    agent=my_agent,
    dataset=eval_dataset,
    evaluators=[
        accuracy_evaluator,
        latency_evaluator,
        LLMJudge("Is the response helpful?"),
    ],
)

LoopGroup

Coordinate multiple loops


group = LoopGroup([
    agent_loop,
    eval_loop,
])

# Health endpoints
# /health/live - liveness probe
# /health/ready - readiness probe

await group.run()

Testing

The testing pyramid

Test Levels

Level Speed What
Prompt Fast Rendering is deterministic
Tool Fast Handler logic
Reducer Fast State transitions
Integration Slow Full agent with model

Example Tests


# Prompt test - no model needed
def test_prompt_renders():
    prompt = MyPrompt(code="def foo(): pass").bind()
    rendered = prompt.render()
    assert "def foo" in rendered

# Reducer test - pure function
def test_reducer():
    reducer = FilesReadReducer()
    op = reducer.on_file_read(FileRead("/a", "content"))
    assert op == SliceOp.append(FileRead("/a", "content"))
